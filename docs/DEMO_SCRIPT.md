# Demo Script

## 60-second walkthrough

1. **Open `/`** — introduce the enterprise layout, pitch-green hero, HTTP 418 badge.

2. **Click "Start Match"** — toss analysis fires. Kettle readiness is 58% (below 65%). Observe the 418 error in the Operations Response console with structured `reasons[]` and `recoveryAction`.

3. **Drag the Kettle Readiness slider to 70%** — watch the Tea Compliance Score update live. The badge changes from WARNING to NOMINAL.

4. **Click "Start Match" again** — now the toss analysis succeeds. HTTP 200, `success: true`.

5. **Switch Tea Mode to "Herbal Infusion"** — score drops by 8 points. Switch to "Masala Chai" — score recovers with a +5 bonus. Tea mode is a first-class compliance variable.

6. **Click "Override Tea Protocol"** — always 418, no matter the tea state. RFC 2324 is non-bypassable. The error code `TEA_BREAK_ENFORCED` appears with the teapot emoji.

7. **Click "Emergency Tea Audit"** — returns compliance score, grade, and full tea state echo. Show the grade field (`NOMINAL`, `OPTIMAL`, etc.).

8. **Navigate to `/demo`** — show the 6-step guided script for judges who want to self-serve.

9. **Close** with: *"Tea governance: the only RFC that's both a joke and a production dependency."*
